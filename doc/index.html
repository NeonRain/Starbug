Introduction

Since web development is such a melange of technologies, the process of actually building a website convolutes the creative and architectural considerations. When you break it down, what is it that we're all really trying to acomplish? I had to start building Starbug to really figure that out. Eventually two processes emerged. Application design and Interaction design. It is my goal to be free to focus on these things while developing with Starbug.

Terminology

Before we move on, let's quickly go over some terms you'll run into.

Models

Models are objects that mediate access to data. They generally correspond to a table in the database and extend from the Table class.

Nouns

Nouns are much like views. They contain HTML and PHP code. They are different from views since there are no controllers in the context of your application.

File Structure

Now let's take a look at Starbug's file structure.

  • /app

    The subfolders of app contain the main code of your application.

    • ./models

      All of your Models will be kept in this folder. An new application will have Users.php, Uris.php and Permits.php in this folder.

    • ./nouns

      This folder contains your applications main resources, from which you can call upon models, utilities and plugins.

  • /public

    This folder contains public resources such as images, stylesheets, and scripts.

  • /plugins

    The plugins folder contains available and installed plugins. Note that some plugins are also in /core/plugins

  • /themes

    Themes provide templates for code generation. Generation is achieved with XSLT.

  • /util

    This folder contains various utilites which can be imported in your application.

  • /script

    This folder contains scripts which you can run from a terminal such as ./script/generate, and ./script/plugin. The corresponding interfaces use these scripts.

  • /core

    This folder contains Starbug's core files.

  • /etc

    As you've hopefully guessed, all configuration files are kept in this folder.

  • /doc

    Here lies documentation.

  • /var

    Various application data may be put in this folder.

Download

You may download Starbug PHP by either checking out a copy from GitHub or dowloading a copy from the website. You may check out a copy as follows:

$ git clone git://github.com/cogentParadigm/Starbug.git starbug

Visit StarbugPHP.com to download a copy.

Once you've checked out a copy, enter the new directory. All commands will be run from this folder.

$ cd starbug

Installation

Once you've downloaded or checked out a copy of Starbug, you need to run the installer in a terminal. The installer will ask you for some configuration information.

$ chmod +x etc/install.php

$ ./etc/install.php

Generating a bare application

Once you have have installed Starbug, you can get started by generating a bare application. You can do this by logging in at http://www.yoursite.com/sb-admin and clicking on generate under Tools, and then choosing Bare Application.

You may also generate it from a terminal like so:

$ ./script/generate app

You will have new files in /app/nouns. The generated files simply rely on public resources from the core such as images and stylesheets. You should put your own in /public.
